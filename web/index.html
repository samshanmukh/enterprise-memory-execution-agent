Critic Agent

AI that validates
its own work

After every research pass, the Critic Agent reviews quality across five dimensions: completeness, accuracy, enterprise relevance, actionability, and balance.

If the confidence score falls below 70, the orchestrator automatically triggers a research retry — passing the critic's specific improvement suggestions back to the Research Agent.

🔁 Automatic Retry Loop

Research → Critic (score < 70) → Research again with feedback → Critic re-scores → Proceed only when approved.

Infrastructure Stack

Built on the right tools
for enterprise AI

Three core platforms — each solving a distinct problem in the agentic AI stack.

🔌

Composio

Unified execution layer for AI agents

Composio is the execution backbone of the entire system. Instead of building and maintaining direct API integrations for each app, Composio provides a single authenticated interface to 200+ business tools — including every app we use.

  • ✓
    One API key, five apps — Notion, Linear, GitHub, Slack, Gmail all managed through Composio's unified toolset
  • ✓
    Managed authentication — OAuth flows, token refresh, and credential storage handled automatically
  • ✓
    Dry-run simulation — realistic mock responses when credentials aren't available, preserving the full trace format
  • ✓
    Retry-safe — idempotent action design means our exponential backoff wrapper works seamlessly
📡

XTrace

Memory infrastructure for AI agents

XTrace is the memory infrastructure layer that sits between AI agents and storage systems. It solves the hardest problem in agentic AI: making agent actions, decisions, and findings durable, queryable, and retrievable across sessions.

  • ✓
    Agent ↔ storage bridge — decouples agents from the underlying storage technology (vector, SQL, or both)
  • ✓
    Persistent context retrieval — semantic similarity search means the agent recalls relevant prior work automatically, without explicit query construction
  • ✓
    Execution tracing — every action is captured with run ID, agent, input, output, status, and duration for full observability
  • ✓
    Cross-session continuity — knowledge accumulates and is versioned, enabling agents to build on previous research instead of starting over
🧈

butterbase

Open-source AI-native backend platform

butterbase is an open-source backend platform built for the AI era. It combines everything an agentic app needs — Postgres, auth, storage, serverless functions, and first-class RAG and MCP support — into one integrated system that works natively with AI agents.

  • ✓
    Postgres + RAG — structured data and vector search in one platform, used for run history and semantic memory
  • ✓
    Durable actors — long-running agent sessions with persistent state that survives restarts and failures
  • ✓
    MCP integration — native support for the Model Context Protocol means butterbase can be exposed directly as an MCP server for any AI agent
  • ✓
    Serverless functions — event-driven triggers that fire when agents produce outputs, enabling downstream automation without polling
Persistent Memory

Remembers.
Recalls.
Compounds.

The Memory Agent maintains two parallel stores — a semantic vector store for similarity-based recall and a structured SQLite store with versioned knowledge records for precise querying.

On every run, the agent retrieves the most relevant prior context before researching. On subsequent runs about the same topic, it builds on its previous work rather than starting over. Knowledge compounds.

🔮

Vector Store (ChromaDB)

Research summaries and key insights stored as embeddings. Queried by cosine similarity — surfaces the most semantically relevant prior context for each new run.

🗄️

Knowledge Records (SQLite)

Structured records with topic, summary, recommendations, sources, critic confidence score, and version number. Supports exact-match and keyword search alongside vector similarity.

📡

Execution Trace (XTrace layer)

Every agent action — with input, output, duration, and status — persisted to JSONL and a relational execution_logs table. Full audit trail for compliance and debugging.

Demo Flow

From prompt to
five deliverables in 10 seconds

Every step is autonomous. No manual clicking. No copy-pasting. No context-switching.

1

Submit a research prompt

POST a single JSON request to the API — or use the Streamlit dashboard. Set dry_run: true to simulate, or false to execute for real.

2

Planner decomposes the request

PlannerAgent calls the LLM and returns a structured 6-step execution plan with typed agent assignments — the blueprint for the entire pipeline.

3

Memory retrieves prior context

MemoryAgent queries ChromaDB by semantic similarity. If this topic has been researched before, the 3 most relevant documents surface automatically — building on prior work.

4

Research Agent synthesises findings

A comprehensive structured analysis is generated — including executive summary, comparison tables, adoption challenges, recommendations, and 4 actionable Linear tasks.

5

Critic Agent validates quality

CriticAgent scores the research 0–100. If confidence falls below 70, the system automatically retries — passing the critic's specific improvement suggestions back as feedback to the Research Agent.

6

Memory stores findings

Research summary and key insights are stored as vector embeddings in ChromaDB, and as a versioned knowledge record in SQLite via the XTrace memory layer — with the critic confidence score attached.

7

Execution Agent fires five Composio actions

In parallel: a Notion page is created, 4 Linear issues are filed, a GitHub issue is opened, a Block Kit Slack message is posted, and a stakeholder Gmail is sent — all with retry logic and full trace logging. Total time: under 10 seconds.

Get Started

Run it in 60 seconds

Clone the repo, copy .env.example, and start the server. No credentials required for dry-run mode.

$ git clone https://github.com/samshanmukh/enterprise-memory-execution-agent
$ pip install -r requirements.txt
$ cp .env.example .env
$ uvicorn app.main:app --reload
